# AI Project Repository Checklist

## Overview

Use this checklist for AI tools, prompt systems, evaluation frameworks, model wrappers, and related projects.

## Required

- [ ] Project purpose and scope are documented.
- [ ] Inputs and outputs are clearly described.
- [ ] Usage examples are provided.
- [ ] Known limitations are documented.
- [ ] License is present.

## Recommended

- [ ] Evaluation approach is explained.
- [ ] Data handling expectations are documented.
- [ ] Safety, quality, or reliability considerations are noted.
- [ ] Versioning or compatibility expectations are defined.
- [ ] Dependencies on external providers or APIs are clear.

## Advanced

- [ ] Benchmarks, scoring, or validation artifacts are included.
- [ ] Observability or logging approach is documented.
- [ ] Responsible-use or governance guidance is included where relevant.
