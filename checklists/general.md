# General Repository Checklist

## Overview

Use this checklist for any repository.

## Required

- [ ] Repository name is clear and appropriate.
- [ ] Repository description is set.
- [ ] `README.md` exists.
- [ ] README explains purpose and scope.
- [ ] License is present when reuse is intended.
- [ ] Repository structure is understandable.
- [ ] Status or level of completeness is reasonably clear.

## Recommended

- [ ] `CONTRIBUTING.md` exists where collaboration is expected.
- [ ] Topics or tags are set on the hosting platform.
- [ ] Basic usage, navigation, or install guidance is included.
- [ ] Metadata is consistent across project files and platform settings.
- [ ] Examples or references are included where helpful.

## Advanced

- [ ] Issue templates or contribution workflows are defined.
- [ ] Changelog or release notes are maintained when relevant.
- [ ] Repository aligns with a formal quality model or publication standard.
