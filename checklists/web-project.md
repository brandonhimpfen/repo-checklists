# Web Project Repository Checklist

## Overview

Use this checklist for websites, static sites, web apps, and front-end projects.

## Required

- [ ] Project purpose is clearly explained.
- [ ] Setup or deployment guidance is included.
- [ ] Folder structure is understandable.
- [ ] License is present where appropriate.
- [ ] Core metadata such as project description and homepage are set where relevant.

## Recommended

- [ ] Accessibility considerations are acknowledged.
- [ ] Responsive behavior is considered.
- [ ] Performance considerations are documented where relevant.
- [ ] Environment or configuration requirements are explained.
- [ ] Screenshots or previews are included when useful.

## Advanced

- [ ] CI checks build or deployment integrity.
- [ ] SEO or metadata guidance is documented for content-driven sites.
- [ ] Contribution or design-system guidance is included for collaborative projects.
