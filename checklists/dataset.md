# Dataset Repository Checklist

## Overview

Use this checklist for data repositories, open datasets, and structured reference data.

## Required

- [ ] Dataset purpose and scope are documented.
- [ ] Data format is explained.
- [ ] Source or provenance is documented.
- [ ] License for the data is clear.
- [ ] Schema or field definitions are provided.

## Recommended

- [ ] Update frequency is documented.
- [ ] Validation rules are defined.
- [ ] Example queries or usage examples are included.
- [ ] File naming is consistent.
- [ ] Known limitations are documented.

## Advanced

- [ ] Automated validation exists.
- [ ] Versioning strategy for data changes is documented.
- [ ] Change history or release notes are maintained.
